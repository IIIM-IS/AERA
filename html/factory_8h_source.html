314 // Goals and predictions:
315 // When positive evidences are found for a goal/prediction, said object is invalidated: in g-monitors and p-monitors, respectively.
316 // Idem for negative evidences In such cases, an absentee (absence of the expected fact) is injected.
317 //
318 // A negative evidence is either:
319 // (a) a fact that is either the absence of the target (absentee) or a fact asserting a state for the target which is not the expected one (it is assumed that an object can be in only one state WRT a given attribute; and recurse in icst) or,
320 // (b) a prediction holding a fact that is a counter evidence for the target with a confidence higher than said target.
321 //
322 // When a decision ground is invalidated, the subsequent is also invalidated: said invalidation is performed in the p-monitors and g-monitors.
323 // When a super-goal is invalidated (simulated or actual), sub-goals are also invalidated: said invalidation is performed in the g-monitors of the sub-goals.
324 //
325 // Invalidation checks are performed at both _take_input() time and reduce() time as the invalidation may occur during the transit in the pipe.
